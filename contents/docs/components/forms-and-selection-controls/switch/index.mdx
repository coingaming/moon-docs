## Behaviour

### Selection

Switch has two selection states:

- Use `unselected` state as default, when the setting is optional and user should enable it themselves or if most users are unlikely to need it on by default.
- Use `selected` state as default, when the setting is expected or recommended for must users, it’s a common practice (such as enable notifications), or if the switch represents a system default setting.

### States

Switch (and all other forms and selection controls) express states. States indicate current condition of the component to someone interacting with it.
Using states ensure clear user feedback, highlighting possible actions or barriers. We have included most common states:

| **State**  | **Context**                                                                                                                                          |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Default`  | Status of the component before the interaction. Also referred to as ‘resting’.                                                                       |
| `Hover`    | When the user moves their cursor over the component, but is not taking the action. Slight visual change can indicate interactivity of the component. |
| `Focus`    | When user selects the component via keyboard navigation, but has not yet taken action on it.                                                         |
| `Disabled` | A non-interactive state where the component appears faded or grayed out. It indicates that the action is unavailable at the moment.                  |

## Size

Dimensions of the switch are tied to the global `scale` tokens. This ensures that the components, especially forms, with similar size settings remain consistent across the interface. `scale` tokens also provides more granular control over dimensional settings.

```css:scale.css showLineNumbers
--scale-height-<size>
--scale-width-<size>
--scale-icon-size-<size>
```
